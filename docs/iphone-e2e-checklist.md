# iPhone実機E2Eチェックリスト

このチェックリストは、GitHub Pages上のPWAをiPhone Safariとホーム画面に追加したPWAの両方で確認するためのものです。PATや記事本文は記録に残さないでください。

## 準備

- [ ] iPhoneのOSバージョンを記録する
- [ ] Safari通常表示で公開URLを開く
- [ ] 同じURLをホーム画面に追加し、PWA表示でも開く
- [ ] 記事リポジトリを読み書きできるfine-grained PATを用意する
- [ ] 画像タスクのある記事と、アップロードする5MB以下のテスト画像を用意する

## 接続と設定

Safari通常表示とPWA表示で、それぞれ確認する。

- [ ] PATを入力して記事一覧が表示される
- [ ] 保存済みトークンの接続テストで、読み取り・書き込み権限の表示を確認できる
- [ ] 期限切れPATまたは権限不足PATで、原因と設定への戻り方が表示される
- [ ] 「次回起動時もこのトークンを使う」を外すと、保存せずに接続できる
- [ ] 設定画面を開いても、保存済みトークンで一覧へ戻れる
- [ ] アプリを閉じて再度開いても、保存済みトークンで接続できる
- [ ] トークン削除後は設定画面に戻り、記事本文が表示されない

## 一覧と記事表示

- [ ] カテゴリタブと記事件数が表示される
- [ ] 記事カードが画面幅内に収まり、横スクロールが発生しない
- [ ] 数字付きファイル名が公開順に並ぶ
- [ ] 記事本文、見出し、箇条書き、リンクが表示される
- [ ] 長い日本語タイトルと本文でレイアウトが崩れない
- [ ] ダッシュボードで公開待ち・レビュー待ち・公開済み・画像未決定の件数が確認できる
- [ ] ファイル名・パス検索と公開状況フィルターを組み合わせて記事を絞り込める
- [ ] 公開スケジュールの開始日時・公開間隔・カテゴリを設定すると予定が表示される
- [ ] 全記事の健全性チェックで、検査中表示・完了結果・問題一覧が表示される

## noteへの転送

- [ ] 「タイトルをコピー」でタイトルがクリップボードへ入る
- [ ] コピー成功後にnote執筆画面またはnoteアプリが開く
- [ ] note側でタイトルを貼り付けられる
- [ ] 記事画面へ戻り、「本文をコピー」で本文をクリップボードへ入れられる
- [ ] note側で本文を貼り付けられる
- [ ] 対応範囲外の表・Markdown画像・raw HTMLがある記事では、対象行と手動対応が表示される
- [ ] 対応範囲外の要素がある場合、note用本文コピーが止まり、原文Markdownへ切り替えられる
- [ ] noteから戻ったとき、元の記事詳細画面が復元される
- [ ] クリップボードが使えない状態では手動コピー用textareaが表示される
- [ ] 手動コピー後の「note執筆画面を開く」からnoteへ進める

## GitHubへの書き戻し

- [ ] 公開済みnote URLを入力して保存できる
- [ ] 一覧へ戻ると対象記事が「公開済み」になる
- [ ] 画像タスクの判断を保存できる
- [ ] 画像を登録すると、記事本文に画像リンクが入り、画像を再表示できる
- [ ] 同じ画像タスクへ別画像を登録すると、画像を安全に置き換えられる
- [ ] 一覧の「画像アセット管理」で、未参照・壊れた参照・状態だけ残った候補を区別できる
- [ ] 未参照画像は選択と確認なしには削除されない

## エラー表示

- [ ] 無効なPATでは、原因を推測できるエラーが表示される
- [ ] 通信を切った状態で、再試行方法が分かる
- [ ] 保存失敗時に、入力内容や記事画面が不必要に消えない

## 記録

確認日、端末、iOS、表示方法（Safari／PWA）ごとに、PASS・FAIL・未確認を記録します。FAILの場合は、操作手順と画面に表示されたメッセージだけを記録し、PATや記事本文は記録しません。

全項目を確認できたら、結果をIssue #1へコメントし、READMEの検証状況とvaultの未検証項目を更新してからIssueをクローズします。
